# x-poster — Complete Build (Single File)

An Eve agent that drafts high-performing X (Twitter) posts. You pick a category, it
researches a timely angle, and returns three ready-to-paste options optimized for the X
"For You" ranking algorithm. You post manually by copy/paste — **no X API needed**.

This one file contains the full project: the structure, the setup steps, and the complete
contents of every file.

---

## Setup

1. **Scaffold the project.** Writes `package.json`, `tsconfig.json`, the `agent/`
   directory, the built-in HTTP channel, and a `dev` script; also runs `git init` and
   installs dependencies.

   ```bash
   npx eve@latest init x-poster
   ```

   It holds the terminal — press **Ctrl+C** once it's running so you can edit files.

2. **Add the OpenAI provider package.**

   ```bash
   cd x-poster
   npm install @ai-sdk/openai
   ```

3. **Create the custom files** below, replacing the generated `agent.ts` and
   `instructions.md`, and adding the skill:

   ```
   agent/agent.ts
   agent/instructions.md
   agent/skills/x_algorithm_playbook.md
   ```

4. **Create your `.env`** and paste in your key:

   ```bash
   cp .env.example .env
   # then edit .env and set OPENAI_API_KEY
   ```

5. **Personalize two things:**
   - In `agent/agent.ts`, set the model string to your OpenAI model.
   - In `agent/instructions.md`, replace the **Categories** list with your own.

6. **Run it.**

   ```bash
   npm run dev
   ```

   Type a message in the terminal UI. It shows the category list, takes your pick,
   researches, and returns three drafts to copy.

### `agent/instructions.md`

```md
# Post Composer — Instructions

You help the user write one excellent X (Twitter) post at a time. The user posts
manually by copy/paste, so your only output is ready-to-paste post text. You never
publish anything and never claim to.

## Flow for every session

1. **Get the category.** If the user hasn't already named one, greet them briefly and
   show the category list below. Ask them to pick a number, or type their own topic.
   Don't draft anything yet.

2. **Research before writing.** Once you have a category, use your web tools to find
   something *timely, specific, and true* — a recent development, a concrete number, a
   named example, or a fresh angle from the last few days. Do not write from memory
   alone. The For You ranker rewards timeliness and punishes vague, generic takes. If
   you can't find anything solid, tell the user instead of inventing something.

3. **Draft three options.** Read the `x_algorithm_playbook` skill, then write THREE
   distinct posts for the chosen category, each taking a different angle:
   - (a) a sharp hook or contrarian claim,
   - (b) a question or prompt that invites replies,
   - (c) a concrete, genuinely useful insight people will want to save or share.

   Each post must be ≤ 280 characters, use 0–1 hashtags, and contain no fabricated
   facts, quotes, or statistics.

4. **Present for copy/paste.** Output the three options, clearly numbered. Under each,
   add one short line naming the main engagement signal it targets (reply / repost /
   profile-click / dwell). Keep the post text itself clean — no preamble or notes baked
   inside the post. End by asking whether they want variations on any option, a
   different angle, or a new category.

## Categories

> These are sensible defaults — replace them with your own. Keep each one specific;
> narrow categories produce sharper posts than broad ones.

1. AI & machine learning
2. Building / shipping in public
3. Startups & founder lessons
4. Developer tools & productivity
5. Tech industry takes & analysis

## Hard rules

- **280 characters max.** Count before showing; trim if over.
- **Never invent.** No made-up facts, quotes, stats, studies, or news. If research is
  thin, say so plainly rather than filling the gap.
- **No bait that backfires.** Avoid rage-bait, fake urgency, or misleading hooks —
  these trigger mutes, blocks, and "not interested," which the ranker weights
  negatively (see the playbook).
- **One idea per post.** Lead with the hook in the first line.
- **Post text only.** Each numbered option is just the post — your commentary goes on
  the separate signal line, not inside the post.
```

### `agent/skills/x_algorithm_playbook.md`

```md
# Writing for the X "For You" ranker

This skill distills how X's open-sourced For You ranking system scores posts, so drafts
can be optimized for reach. Source: the public x-algorithm repository (the home-mixer /
Phoenix ranking pipeline).

## How scoring works

The ranker predicts the probability of many different engagement actions for each post,
then combines them into one score:

    Final Score = Σ ( weight_i × P(action_i) )

There is no keyword matching and no hashtag boosting. Relevance is learned from real
engagement, so what the post *makes people do* is what moves the score.

## Positive signals — write to earn these

These actions carry positive weight. Posts that reliably cause them rank higher:

- **P(reply)** — invite a response: a question, a take people will want to correct, a
  fill-in-the-blank, a "what would you add?"
- **P(repost) / P(quote) / P(share)** — make it worth re-broadcasting: a crisp insight,
  a useful list, a surprising-but-true fact.
- **P(profile_click) / P(follow_author)** — give people a reason to want more from you. A
  specific, high-signal point makes the next click feel worth it.
- **P(dwell)** — reward time-on-post: a complete, self-contained thought beats a vague
  tease. Make them stop and read, not bounce.
- **P(favorite) / P(click) / P(video_view) / P(photo_expand)** — supporting signals; a
  clear payoff and a clean hook lift all of them.

## Negative signals — these suppress the post

These carry negative weight. One post that triggers them can drag down the whole account:

- **P(not_interested)**, **P(mute_author)**, **P(block_author)**, **P(report)**

Avoid: rage-bait, fake urgency, misleading hooks, spammy repetition, and anything that
feels like clickbait the reader regrets. The model is explicitly built to push down
content people would dislike.

## Structural facts to work with

- **Author diversity is attenuated.** Repeated posts from the same author get scored down
  for feed diversity, so spacing posts out matters more than posting in bursts.
- **Timeliness helps.** Old posts are filtered out before ranking — fresh, current takes
  have an edge. This is why the research step looks for *recent* angles.
- **Candidates are scored in isolation.** A post's score doesn't depend on what else is in
  the feed, so each post has to stand on its own merits.
- **No hand-engineered features.** Hashtags, keyword stuffing, and formatting tricks do
  little. Substance, hooks, and replies do the heavy lifting.

## Quick checklist before finalizing a post

1. Does the first line hook in under a second?
2. Is there a clear reason to reply, repost, or click the profile?
3. Is every fact real and current?
4. Could it provoke a mute / block / report? If yes, soften or cut.
5. One idea, ≤ 280 characters, 0–1 hashtags.
```

## How it works

- **Model** — `agent/agent.ts` uses `openai(...)` from `@ai-sdk/openai`, so your
  `OPENAI_API_KEY` is used directly rather than routing through the Vercel AI Gateway.
- **Behavior** — `agent/instructions.md` is the system prompt. It defines the
  category → research → draft → copy/paste flow.
- **Research** — handled by Eve's built-in web tools (part of the default harness). No
  custom tool needed.
- **Reach** — `agent/skills/x_algorithm_playbook.md` is loaded when drafting and encodes
  what the For You ranker rewards and penalizes.

## Notes

- This agent **never posts**. It only produces text for you to paste.
- Keep within X's automation and content rules when you post.
- Once installed, the eve package bundles its full docs at `node_modules/eve/docs` — check
  there to confirm the exact built-in web tool names if you want to reference them
  explicitly in `instructions.md`.
